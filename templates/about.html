Model Architecture & Technical Details

Base Architecture

  • Transfer Learning: EfficientNetB0 as feature extractor
  • Custom Top Layers: GlobalAveragePooling2D → Dense (128 , ReLU) → Dropout (0.3) → Dense (64 , ReLU) → Output (8 neurons, softmax)
  • Fine-tuning: Last 40 layers unfrozen for specialized training

Training Configuration

  • Optimizer: Adam with learning rate 5e-5
  • Class Weights: Applied to handle class imbalance
  • Early Stopping: Applied with Patience 7
  • Reduce LR: Reduced LR on Plateau

Dataset Information

  • Total Images: 6,000 Image samples
  • Training: 80% for Training
  • Validation: 10% for Validation
  • Testing: 10% for Test
  • Resolution: 224×224 px (preprocessed)

Key Challenges & Solutions

Class Imbalance

  • Imbalanced data with rare classes
  • As a solution, applied class weighting during training
  • Found better than data augmentation

Hyperparameter Tuning

  • Extensive search for optimal learning rate
  • Experimentation with different dropout rates
  • Early stopping to prevent overfitting

Transfer Learning

  • Leveraged EfficientNetB0 pre-trained on ImageNet
  • Unfroze some layers during fine-tuning
  • Custom top layers for blood group specialization

Conclusion

Our blood group classification model demonstrates state-of-the-art performance in identifying blood types. By leveraging transfer learning with EfficientNetB0 and implementing strategic solutions to handle class imbalance, we've created a robust system with 90% test accuracy.

The model shows particular strength in identifying common blood types (A+, O+, B+) while maintaining good performance on rarer types. Our confusion matrix reveals that most misclassifications occur between similar blood types, primarily due to subtle visual differences that challenge even expert human analysts.

Future improvements will focus on expanding the dataset for rare blood types and exploring ensemble methods to further enhance the model's reliability for clinical applications.
